# Architecture Decision Records (ADR)

Decisões arquiteturais **significativas** versionadas no repositório.

## Quando criar um ADR

- Escolha de stack ou framework
- Mudança de padrão arquitetural
- Trade-off com impacto de longo prazo
- Decisão que a equipe não deve re-litigar

## Quando NÃO criar um ADR

- Decisões operacionais do dia → gstack `decisions.jsonl`
- Detalhes de implementação triviais
- Anything re-derivável do código

## Formato

Usar template MADR (configurado no ruflo). Nomear arquivos:

```
NNNN-titulo-curto-em-kebab-case.md
```

Exemplo: `0001-usar-postgresql-como-banco-principal.md`
